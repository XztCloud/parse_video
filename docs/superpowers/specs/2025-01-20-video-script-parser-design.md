# 视频脚本解析平台 - 需求规格文档

## 项目概述

开发一个Web应用，用于解析视频内容并生成结构化脚本（台词、镜头描述、时间戳），帮助用户分析爆款视频的创作模式，快速制作同款视频。

## 核心功能

### 1. 视频输入
- **本地上传**：支持上传 MP4/MOV/AVI 等常见视频格式，单文件上限 500MB
- **抖音链接解析**：输入抖音分享链接，自动提取视频并下载到服务器进行解析

### 2. 视频脚本生成
- **台词提取**：通过语音识别（ASR）提取视频中所有语音内容，按时间戳分段
- **镜头分析**：通过视觉理解模型分析视频画面变化，识别镜头切换、场景描述
- **时间戳标注**：每段台词和镜头均带有精确时间戳（精确到秒）
- **脚本结构化输出**：生成包含时间戳、镜头描述、台词的完整脚本

### 3. 脚本展示与管理
- **脚本详情页**：以时间轴形式展示脚本，支持逐段查看
- **脚本编辑**：用户可手动修正AI生成的脚本内容
- **脚本导出**：支持导出为 TXT / SRT / JSON 格式
- **历史记录**：保存用户解析过的视频和脚本，支持回看

### 4. 同款视频参考
- **脚本模板化**：将解析后的脚本转化为可复用的创作模板
- **关键要素提取**：自动标注视频的节奏、镜头语言、文案结构等关键创作要素

## 技术方案

### AI服务选型（国内云端）
- **语音识别（ASR）**：阿里云智能语音交互（Paraformer模型）—— 中文语音识别准确率业界领先，支持长音频
- **视觉理解**：通义千问 VL（Qwen-VL）—— 支持视频帧分析，镜头理解能力强
- **文本整理**：通义千问（Qwen-Plus）—— 将ASR原始输出和视觉分析结果整理为结构化脚本

### 前端技术栈
- **框架**：Vue 3 + TypeScript
- **UI组件库**：Element Plus
- **构建工具**：Vite
- **视频播放**：video.js
- **时间轴组件**：自定义开发

### 后端技术栈
- **框架**：Python FastAPI
- **任务队列**：Celery + Redis（视频解析为耗时任务，需异步处理）
- **数据库**：PostgreSQL（结构化数据）+ MinIO/本地存储（视频文件）
- **视频处理**：FFmpeg（抽帧、音频提取、格式转换）
- **抖音解析**：yt-dlp（支持抖音平台视频下载）

## 系统架构

```
用户浏览器 (Vue3)
    │
    ├── 上传视频 / 输入抖音链接
    │
    ▼
FastAPI 后端
    │
    ├── 视频接收 → 存储到 MinIO/本地
    │
    ├── 创建解析任务 → Celery 异步队列
    │
    ├── Celery Worker:
    │   ├── FFmpeg 提取音频 → 阿里云ASR → 台词文本
    │   ├── FFmpeg 抽帧 → 通义千问VL → 镜头描述
    │   └── 通义千问 → 整合结构化脚本
    │
    ├── WebSocket/SSE → 前端推送解析进度
    │
    └── 脚本存储 → PostgreSQL
```

## 数据模型

### Video（视频）
- id, user_id, title, source_type(local/douyin), source_url, file_path, duration, status(pending/processing/done/failed), created_at

### Script（脚本）
- id, video_id, content(JSON), raw_asr_text, raw_visual_text, created_at, updated_at

### ScriptSegment（脚本片段）
- id, script_id, start_time, end_time, shot_description, dialogue, segment_type(shot/dialogue/mixed)

## 用户流程

1. 用户上传视频或输入抖音链接
2. 系统接收视频，创建解析任务
3. 前端通过SSE实时显示解析进度（提取音频→识别台词→分析镜头→生成脚本）
4. 解析完成后，用户查看结构化脚本
5. 用户可编辑、导出脚本，或将其保存为创作模板

## 非功能需求

- 视频解析任务支持并发，单用户最多3个并行任务
- 解析进度实时推送（SSE）
- 视频文件存储支持本地磁盘（开发阶段）和MinIO（生产阶段）
- API接口遵循RESTful规范
